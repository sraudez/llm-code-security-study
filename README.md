# Final Project: LLM-Generated C Security Analysis

This repository benchmarks security quality in C code generated by LLMs across a fixed set of programming tasks.  
It includes:
- prompt definitions per task,
- generated C outputs from multiple models,
- static analysis tooling (Semgrep + cppcheck),
- and manual review guidance/results.

## Project Layout

```text
final_project/
├── run_analysis.py
├── manual_review.md
├── tasks/
│   ├── task01_read_username/
```

## Data Organization

Each `tasks/taskXX_*` directory contains:
- `prompts.md` with 3 prompt types:
  - `minimal`
  - `detailed`
  - `secure`
- model output folders (for example `ChatGPT/` and `Claude/`)
- generated files named like:
  - `minimal_1.c`, `minimal_2.c`
  - `detailed_1.c`, `detailed_2.c`
  - `secure_1.c`, `secure_2.c`

## Prerequisites

Install required CLI tools:

- [Semgrep](https://semgrep.dev/docs/getting-started/quickstart)
- [cppcheck](http://cppcheck.sourceforge.net/)

Examples:

```bash
pip install semgrep
brew install cppcheck
```

Python requirements:
- Python 3
- `pandas` (for manual-review summary script)

```bash
pip install pandas
```

## Running Static Analysis

From the repository root:

```bash
python run_analysis.py
```

This script:
1. Verifies `semgrep` and `cppcheck` are installed.
2. Scans all `*.c` files under `tasks/`.
3. Runs Semgrep (`p/c` ruleset) and cppcheck per file.
4. Writes CSV outputs in `analysis/`.

### Output Files

- `analysis/semgrep_results.csv`: one row per Semgrep finding
- `analysis/cppcheck_results.csv`: one row per cppcheck finding
- `analysis/summary.csv`: one row per scanned file with aggregate flags/counts

## Summarizing cppcheck Findings by Rule

```bash
python analysis/summarize_cppcheck_by_rule.py
```

Optional flags:

```bash
python analysis/summarize_cppcheck_by_rule.py \
  --input analysis/cppcheck_results.csv \
  --output analysis/cppcheck_summary_by_rule.csv
```

Print-only mode:

```bash
python analysis/summarize_cppcheck_by_rule.py --no-write
```

## Summarizing Manual Review Results

If you have `analysis/manual_review.csv` populated:

```bash
python analysis/summarize_manual_results.py
```

This prints grouped counts by:
- prompt type
- model
- severity
- task

## Manual Review Protocol

Use `manual_review.md` for reviewer guidance, including:
- common security checks (input validation, memory management, unsafe APIs),
- task-specific checks for all 10 tasks,
- severity policy (`HIGH`, `MEDIUM`, `LOW`).

## Important Notes

- `run_analysis.py` currently expects model folder names `Chatgpt` and `Claude`.
- If your directory uses `ChatGPT` (uppercase `GPT`), update either:
  - `VALID_MODELS` in `run_analysis.py`, or
  - the folder name in `tasks/*/`.
- Run scripts from the repository root so relative paths resolve correctly.

## Reproducibility Checklist

1. Confirm task folders and generated `.c` files are present.
2. Install Semgrep + cppcheck + Python dependencies.
3. Run `python run_analysis.py`.
4. (Optional) Run `python analysis/summarize_cppcheck_by_rule.py`.
5. (Optional) Populate `analysis/manual_review.csv` and run `python analysis/summarize_manual_results.py`.
